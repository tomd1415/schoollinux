Overview of Phase 3

At this point, you have set up and configured the 20 classroom machines to authenticate against Active Directory, mount user home directories from the central file server, and provide OneDrive access through Rclone. In Phase 3, you will:

  • Introduce a group of pupils or test accounts to the new environment.
  • Monitor their experience as they log in, access files, and use the system under normal classroom conditions.
  • Collect feedback on usability, performance, and reliability.
  • Identify and fix any minor issues discovered through real-world testing.

By the end of Phase 3, you should have a clear understanding of how the system performs under actual use and what, if any, adjustments are required before moving on to the evaluation phase.

Step 1: User Logins and Experience

Begin by having a small group of pupils (or a few test accounts) log into multiple machines. Observe their interactions and note any issues. The goal is to confirm that domain authentication works smoothly and consistently.

Key Tasks:

  • Instruct test users to log in using their AD credentials on various machines in the test environment.
  • Confirm that login times are reasonable and users are not experiencing slow authentication.
  • Observe if home directories mount correctly for each user and that their initial files (if any) are accessible.

Reference: Samba Domain Member Guide

Step 2: Home Directories and OneDrive Access

Verify that user home directories and OneDrive integration are functioning as intended. Users should be able to access their files from any machine, and their OneDrive files should be available on demand without significant delay.

Key Tasks:

  • Ask users to open their home directories on multiple machines, ensuring the same files are visible each time.
  • Have users open OneDrive from their home directory mount point and verify that files list and load correctly.
  • Test file creation, editing, and saving back to OneDrive. Confirm that changes persist and sync as expected.

Reference: Rclone OneDrive Documentation

Step 3: Performance and Reliability Checks

Assess how the system performs under typical classroom usage. Multiple users logging in simultaneously, accessing various files, and using their home directories should not degrade performance significantly.

Key Tasks:

  • Monitor login times as several users log in at once. Are logins still quick and stable?
  • Check network file transfers: have users open, edit, and save multiple small files and a few large files.
  • Note any latency or slowdowns when accessing OneDrive files. Are they loading quickly enough for practical classroom use?

Reference: Debian Performance Tuning

Step 4: Feedback and Issue Collection

Gather qualitative and quantitative feedback from users. If pupils find certain operations confusing, or if staff notice performance issues, record these observations for future adjustments.

Key Tasks:

  • Conduct a brief survey or ask pupils directly about their experience. Note usability issues, slow performance, or confusion.
  • Keep track of any error messages or unexpected behaviours that occur during normal use.
  • Document suggestions from staff or pupils on how to improve file access, login procedures, or general user experience.

Step 5: Refinements and Minor Adjustments

Use the collected feedback to make minor adjustments to configurations. Update Ansible playbooks, tweak Samba or Winbind settings, or adjust Rclone mounts as needed.

Key Tasks:

  • If login times are slow, consider adjusting caching settings in Winbind or reviewing network bandwidth.
  • If OneDrive access is sluggish, explore Rclone caching options or consider evaluating network throughput.
  • Update and rerun your Ansible playbooks to deploy any configuration changes across all 20 machines.

Reference: Ansible Documentation
