Overview

In this phase, you will prepare the infrastructure and environment for the pilot program. You are using Debian Linux and relying on open-source solutions. The goal is to ensure that when you move to implementation in the next phases, everything is ready and configured according to best practices.

Step-by-Step Instructions

Complete each task in order. Tick each box as you finish. Use the comment box at the end to record notes.

1. Network Isolation and VLAN Setup

Set up a dedicated VLAN or subnet for the 20 test machines. This ensures that the pilot environment does not interfere with the main school network. Consider using your network switches' configuration interface or consult your network documentation for instructions.

Why: Isolation improves security, manageability, and prevents unintended conflicts.

How:

  • Configure a separate VLAN/subnet on your school’s core switch (e.g. VLAN 200, subnet 192.168.200.0/24).
  • Ensure DHCP is available on this VLAN to simplify client setup.
  • Test connectivity from one test machine to ensure it can reach the file server and the internet.

Reference: Debian Network Configuration Guide

2. Central File Server Setup (Samba/NFS)

Prepare a central file server to host shared home directories and user profiles. For integration with Microsoft Active Directory, Samba is often the best choice. NFS can be used for non-AD integrated shares if preferred.

Why: Centralizing home directories allows users to have a consistent environment across all 20 machines.

How:

  • Install Samba on a dedicated Debian server: sudo apt update && sudo apt install samba winbind libpam-winbind libnss-winbind
  • Configure /etc/samba/smb.conf for your AD realm and ensure security = ads.
  • Join the server to the AD domain: sudo net ads join -U Administrator, then test with wbinfo -u
  • Set up [homes] share in Samba to serve user home directories.
  • (Optional) If using NFS for non-AD shares: sudo apt install nfs-kernel-server and configure /etc/exports
  • Test file access: From a test client, run smbclient -L //fileserver and ensure shares are visible.

Reference: Samba Wiki

3. Debian Installation Media and Base Configuration

Prepare installation media for Debian and ensure that you have a standard base image ready. This will make deploying 20 identical systems easier.

Why: Consistency in deployment ensures fewer configuration errors down the line.

How:

  • Download the latest stable Debian ISO from the official site: Debian Downloads
  • Create bootable USB sticks for each machine or set up a network-based installation server (PXE boot).
  • Perform a test installation on one machine, ensuring you can connect to the VLAN and file server.

4. Planning Ansible Configuration

You’ll be using Ansible to automate configurations. Ensure Ansible is installed and you have a basic inventory file for the 20 machines.

Why: Ansible automation saves time, reduces manual errors, and ensures consistent setups.

How:

  • sudo apt install ansible on your control machine.
  • Create an Ansible inventory file inventory.ini listing your 20 test machines by hostname or IP.
  • Set up SSH key-based authentication from your control machine to each test machine.

Reference: Ansible Documentation

5. Verification and Initial Testing

Before moving on to Phase 2, verify that your infrastructure is stable and all prerequisites are met.

Why: Detecting and fixing issues now prevents bigger problems later.

How:

  • Confirm VLAN isolation: Machines in the VLAN should not see the main school network resources unless intended.
  • Confirm Samba domain join: wbinfo -u lists domain users.
  • Test file transfers: Create a test file in a home directory on the server and ensure a client can read it.
  • Ensure Ansible can ping all machines: ansible -i inventory.ini all -m ping
