Overall Project-Level Objectives

These high-level objectives span the entire project and connect to broader computing concepts and professional practices.

  • Network and Systems Understanding: Understand networks, servers, and authentication systems.
  • Linux and Open-Source Competency: Work confidently with Linux and open-source tools.
  • Configuration Management and Automation: Use tools like Ansible for consistent deployments.
  • Performance and Usability Considerations: Evaluate system performance and user feedback.
  • Documentation and Professional Practices: Apply version control, documentation, and reporting.

Phase-Level Objectives

Phase 1: Preparation

  • Network Segmentation and DHCP: Implement VLANs/subnets and configure DHCP.
  • Centralised File Services: Set up Samba/Winbind integrated with AD.
  • Debian Installation and PXE: Understand PXE booting and Debian installs.
  • Configuration Management (Ansible): Set up Ansible for automation.
  • Verification and Troubleshooting: Verify settings and resolve issues.

Phase 2: Implementation

  • OS Deployment: Uniformly deploy Debian using PXE.
  • Package Installation & Configuration (Ansible): Install and configure essential packages.
  • AD Domain Integration: Join Linux clients to the AD domain.
  • User Home Directories & PAM: Configure PAM for automatic home creation.
  • Cloud Integration (OneDrive): Use Rclone for OneDrive access.
  • Verification Tests: Test configurations for performance and reliability.

Phase 3: Testing

  • User Experience Testing: Connect configurations to practical classroom use.
  • Home Directories & OneDrive Validation: Ensure consistency and cloud access.
  • Performance & Reliability Checks: Assess performance under load.
  • Feedback & Issue Collection: Gather user input and identify improvements.
  • Refinements & Adjustments: Apply minor changes to address issues.

Phase 4: Evaluation

  • Feedback Analysis & Prioritisation: Interpret user feedback and prioritise fixes.
  • Performance & Log Review: Compare logs and metrics against benchmarks.
  • Decision Making & Next Steps: Decide on wider rollout or further refinements.
  • Documentation & Training: Update docs and provide training resources.
