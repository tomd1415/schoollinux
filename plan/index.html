Project Overview

This pilot project aims to transition a classroom of 20 machines to Debian Linux, integrated with Microsoft Active Directory (AD) and using open-source tools for file access. Each machine will authenticate against AD, mount centrally hosted home directories, and provide access to cloud storage (OneDrive) via open-source solutions. The environment will be managed and configured through automation (e.g. Ansible) for consistency.

Phases of the Pilot

Each phase builds on the previous one, ensuring a structured approach to deploying, testing, and evaluating the new environment.

Phase 1: Preparation

Goal: Set the foundation for the pilot by isolating the test environment, preparing servers, and establishing automation frameworks.

Key Steps Include:

  • Configure a dedicated subnet (192.168.8.0/24) and VLAN for the 20 pilot machines.
  • Set up a central file server using Samba/Winbind integrated with AD.
  • Use a PXE server (192.168.8.3) to install Debian on all machines from the same image.
  • Install and configure Ansible on a control machine to manage all 20 Debian systems.
  • Verify basic connectivity, domain integration, and file share visibility before proceeding.

View Detailed Instructions for Phase 1

Phase 2: Implementation

Goal: Deploy Debian to all machines, integrate them with AD, and configure them for centralized home directories and OneDrive access.

Key Steps Include:

  • Use PXE booting and the prepared Debian image to install the OS on all 20 machines.
  • Leverage Ansible to install and configure Samba, Winbind, Rclone, and related packages.
  • Join each machine to the AD domain and confirm domain user lookup works consistently.
  • Set up PAM configurations so that home directories are created and mounted on login.
  • Integrate Rclone-based OneDrive access, ensuring users can access their cloud files easily.

View Detailed Instructions for Phase 2

Phase 3: Testing

Goal: Validate the environment by having test users log in, test file access, and confirm that everything performs as expected.

Key Steps Include:

  • Have a subset of pupils or test accounts log in to multiple machines using their AD credentials.
  • Verify that home directories follow each user regardless of which machine they use.
  • Check OneDrive file access through Rclone mounts, ensuring stable file loading.
  • Monitor performance with both large and small file transfers.
  • Gather feedback from users on usability, performance, and reliability.

View Detailed Instructions for Phase 3

Phase 4: Evaluation

Goal: Analyse the results of the pilot and determine whether to scale the solution further.

Key Steps Include:

  • Review feedback from pupils and staff to identify strengths and weaknesses.
  • Evaluate system logs, performance metrics, and user experience.
  • Decide if the pilot environment meets educational and technical goals.
  • Plan for documentation updates, training, or additional tools before a full rollout.
